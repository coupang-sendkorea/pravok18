# Право CRM — обновление интерфейса и финансового учета

В этой версии добавлено:

- прокрутка в блоках **Напоминания по оплатам**, **Ближайшие события**, **Касса и операции** после двух записей;
- боковое меню закреплено сразу под названием **Право CRM**;
- касса разделена на **наличный** и **безналичный** расчет;
- добавлен раздел **Займы**;
- добавлен раздел **Личные расходы и расчеты** для **Ильвара** и **Рустама**;
- уменьшен размер шрифта и сделан более компактный интерфейс.

## Что нужно обновить

На GitHub замените файлы:

- `index.html`
- `app.js`
- `styles.css`
- `supabase-schema.sql`

## Что нужно сделать в Supabase

1. Откройте **SQL Editor**.
2. Откройте файл `supabase-schema.sql`.
3. Скопируйте весь текст.
4. Вставьте в SQL Editor.
5. Нажмите **Run**.

Это обязательно, потому что добавлены:

- поля каналов оплаты в `payments` и `expenses`;
- новые поля в `cash_transactions`;
- таблица `loans`;
- таблица `personal_records`.

## Как работает логика

### Касса

Общий баланс считается из:

- оплат по договорам;
- расходов конторы;
- иных операций;
- полученных займов;
- погашений займов;
- личных расчетов, где было движение денег.

### Займы

- при создании займа он отражается как поступление в кассу;
- погашение займа делается через кнопку **Погасить** или через обычную операцию с назначением **Погашение займа**;
- остаток займа считается автоматически.

### Личные расходы

Доступны записи:

- личные деньги потрачены на компанию;
- компания погасила личный расход;
- сотрудник взял деньги компании на личные нужды;
- сотрудник вернул деньги компании.

По каждому человеку показывается:

- сколько компания должна ему;
- или сколько он должен компании.
