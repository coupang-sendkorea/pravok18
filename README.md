# Право CRM

Готовая статическая CRM для юристов на **GitHub Pages + Supabase**.

## Что умеет программа

- хранить клиентскую базу;
- открывать карточку клиента по нажатию на ФИО;
- сохранять контакты клиента;
- вести сумму договора;
- учитывать способы оплаты:
  - 100% предоплата;
  - частями;
  - по графику;
- хранить срок оплаты и дату напоминания;
- автоматически считать:
  - фактически оплачено;
  - остаток долга;
  - дополнительные расходы конторы;
- вести календарь событий на главной странице;
- показывать ближайшие события и напоминания по оплате;
- отправлять дело в архив кнопкой **«Дело завершено»**;
- возвращать дело из архива.

## Важный нюанс по безопасности

В текущей версии вход по паролю `pravo888` реализован как **парольный экран перед интерфейсом**. Для статического сайта на GitHub Pages это удобно, но **не является полноценной защищённой авторизацией**.

Если вам потребуется настоящая защита данных, следующим этапом лучше перевести вход на **Supabase Auth**.

## Пошаговый запуск

### Шаг 1. Создайте проект в Supabase

1. Откройте Supabase Dashboard.
2. Создайте новый проект.
3. Дождитесь, пока проект полностью запустится.

### Шаг 2. Создайте таблицы

1. В Supabase откройте **SQL Editor**.
2. Откройте файл `supabase-schema.sql`.
3. Скопируйте весь SQL.
4. Вставьте его в SQL Editor.
5. Нажмите **Run**.

После этого создадутся таблицы:

- `clients`
- `payments`
- `expenses`
- `payment_schedules`
- `events`

### Шаг 3. Скопируйте ключи проекта

В Supabase откройте **Connect** или **Project Settings → API Keys** и скопируйте:

- `Project URL`
- `Publishable key` (или legacy `anon key`, если используете старый формат)

### Шаг 4. Заполните файл `config.js`

Откройте файл `config.js` и вставьте свои значения:

```js
window.APP_CONFIG = {
  SUPABASE_URL: 'https://ВАШ-ПРОЕКТ.supabase.co',
  SUPABASE_KEY: 'ВАШ_PUBLISHABLE_ИЛИ_ANON_KEY',
  APP_PASSWORD: 'pravo888',
  APP_TITLE: 'Право CRM'
};
```

### Шаг 5. Создайте репозиторий на GitHub

Создайте новый репозиторий, например:

- `law-crm`
- `pravo-crm`
- `office-crm`

### Шаг 6. Загрузите файлы в репозиторий

Загрузите в корень репозитория эти файлы:

- `index.html`
- `styles.css`
- `app.js`
- `config.js`
- `.nojekyll`

`README.md` и `supabase-schema.sql` тоже можно загрузить — это удобно для хранения инструкции и SQL.

### Шаг 7. Включите GitHub Pages

1. Откройте репозиторий на GitHub.
2. Перейдите в **Settings**.
3. Откройте раздел **Pages**.
4. В блоке **Build and deployment** выберите:
   - **Source** → **Deploy from a branch**
   - **Branch** → `main`
   - **Folder** → `/ (root)`
5. Нажмите **Save**.

Через некоторое время GitHub выдаст ссылку на сайт.

### Шаг 8. Откройте сайт

1. Перейдите по ссылке GitHub Pages.
2. Введите пароль:

```text
pravo888
```

3. После входа программа загрузит данные из Supabase.

## Как пользоваться

### 1. Добавление клиента

- Нажмите **«Новый клиент»**.
- Заполните ФИО, дело, контакты.
- Укажите сумму договора.
- Выберите способ оплаты.
- Укажите срок оплаты и дату напоминания.
- Сохраните карточку.

### 2. Добавление фактической оплаты

- Откройте карточку клиента.
- В блоке **«Фактические оплаты»** нажмите **«Добавить оплату»**.
- Укажите дату, сумму и комментарий.
- Сохраните карточку.

После этого:

- сумма договора останется прежней;
- сумма оплаченного увеличится;
- сумма долга уменьшится автоматически.

### 3. Добавление расходов

- В карточке клиента откройте блок **«Дополнительные расходы конторы»**.
- Добавьте почтовые расходы, госпошлину и другие затраты.

### 4. График оплат

- В блоке **«График оплат»** добавьте этапы.
- Укажите срок, сумму, статус и комментарий.

### 5. События и календарь

- На главной странице есть календарь.
- Для создания события нажмите **«Новое событие»**.
- Можно привязать событие к клиенту.
- Ближайшие события выводятся на главной странице отдельным блоком.

### 6. Завершение дела

- Откройте карточку клиента.
- Нажмите **«Дело завершено»**.
- Дело уйдет в архив.

### 7. Возврат из архива

- Перейдите в раздел **«Архив»**.
- Нажмите **«Вернуть»** у нужного дела.

## Если нужно поменять валюту

Сейчас валюта в интерфейсе настроена на **RUB**.

Если нужна другая валюта, откройте `app.js`, найдите строку:

```js
currency: 'RUB'
```

и замените на нужное значение, например:

- `EUR`
- `USD`
- `KZT`

## Совет по реальной защите данных

Для тестового запуска текущая версия подходит.

Для рабочего юридического офиса лучше сделать следующий этап:

- вход через email + пароль в Supabase Auth;
- закрытые RLS-политики;
- приватный доступ только вашим сотрудникам.
